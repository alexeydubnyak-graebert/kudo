# Настройка проекта ARES kudo

## Конфигурация путей к иконкам

Проект использует корпоративный репозиторий с иконками `application-icons`. Путь к репозиторию настраивается через переменную окружения.

### Первоначальная настройка

#### Автоматическая настройка (рекомендуется)

Запустите скрипт автоматической настройки:

```bash
./scripts/setup-env.sh
```

Скрипт создаст `.env` файл с путем `~/Documents/GitHub/application-icons`, который автоматически работает для любого пользователя.

#### Ручная настройка

1. Скопируйте файл `.env.example` в `.env`:

   ```bash
   cp .env.example .env
   ```

2. Файл уже содержит правильную конфигурацию с использованием `~/`:
   ```env
   VITE_ICONS_PATH=~/Documents/GitHub/application-icons
   ```

Символ `~` автоматически заменяется на домашнюю директорию текущего пользователя!

### Варианты конфигурации

#### С использованием ~/ (рекомендуется)

```env
VITE_ICONS_PATH=~/Documents/GitHub/application-icons
```

✅ Работает на любом компьютере любого пользователя!

#### Относительный путь

```env
VITE_ICONS_PATH=../../../application-icons
```

#### Абсолютный путь

```env
VITE_ICONS_PATH=/Users/alexey/Documents/GitHub/application-icons
```

⚠️ Нужно менять для каждого пользователя

### Структура репозитория application-icons

Репозиторий должен иметь следующую структуру:

```
application-icons/
├── ribbon/
│   └── home/
│       └── draw/
│           ├── light/
│           │   ├── line.svg
│           │   ├── circle.svg
│           │   └── ...
│           └── dark/
│               ├── line.svg
│               ├── circle.svg
│               └── ...
```

### Работа в команде

- Файл `.env` добавлен в `.gitignore` и не коммитится в репозиторий
- Каждый разработчик создает свой `.env` файл с путем к `application-icons` на своем компьютере
- Файл `.env.example` содержит пример конфигурации и коммитится в репозиторий

### Проверка конфигурации

После настройки `.env` файла перезапустите dev-сервер:

```bash
npm run dev
```

Иконки должны загружаться из указанного репозитория.

## Troubleshooting

### Иконки не отображаются

1. Проверьте, что путь в `.env` правильный
2. Убедитесь, что репозиторий `application-icons` склонирован
3. Проверьте структуру папок в `application-icons`
4. Перезапустите dev-сервер после изменения `.env`

### Разные пути на разных компьютерах

Это нормально! Каждый разработчик использует свой `.env` файл с путем, соответствующим его системе.
